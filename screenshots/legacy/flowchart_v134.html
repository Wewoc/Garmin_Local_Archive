System Architecture Flowchart

Data flow from API ingestion through schema validation and transformation to the self-healing revalidation loop triggered on schema version mismatch.

01 — Ingestion

Raw persistence immediately after API fetch. Foundation for revalidation without new API calls. garmin_writer.py is the sole owner of raw/ and summary/.

02 — Orchestration

garmin_collector.py coordinates all modules. No domain logic — pure flow control. Triggers schema version check on every process start.

03 — Validation ✦ v1.3.4

Structural schema enforcement. No hard stop — degraded mode only. required missing → critical, optional missing → warning. Pipeline always continues.

04 — Assessment

garmin_quality.py evaluates content (not structure). Receives validator_result + validator_issues. Writes three new fields per day to quality_log.json.

05 — Self-Healing

On schema version mismatch: days with result ≠ ok are re-read from raw/ and revalidated. No API call. Quality only re-evaluated if the result changes.
