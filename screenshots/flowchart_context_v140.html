v1.4.0 — CONTEXT PIPELINE

Context Pipeline Flowchart

External data collection via Open-Meteo API — weather and pollen data fetched per location segment and written to local context_data/. No live API calls at dashboard build time.

01 — Trigger

Initiated by the GUI "Sync Context" button. Runs in a background thread. Cancellable via stop_event at any point.

02 — Orchestration

context_collector.py reads the date range from the local Garmin archive and splits it into segments per location row in local_config.csv.

03 — Plugins

weather_plugin.py and pollen_plugin.py are metadata only — no executable logic. They define endpoints and fields. New plugins added to _PLUGINS list in context_collector.

04 — Fetch

context_api.fetch() calls Open-Meteo per plugin per segment. Returns (raw, failed_endpoints). No API key required. Degraded mode on partial failure.

05 — Write

context_writer.py is sole owner of context_data/. Writes raw JSON per plugin per segment. Never called from dashboard pipeline — read-only there.
