v1.4.0 — DASHBOARD PIPELINE

Dashboard Pipeline Flowchart

From GUI trigger through auto-discovery and specialist execution to rendered output. Data brokers (maps/) route requests to local archives — no live API calls at build time.

01 — Trigger

GUI "Create Reports" opens a checkbox popup. Selection passed to dash_runner.build(). garmin_config reloaded before build — config cached in same process.

02 — Discovery

dash_runner.py scans dashboards/ for *_dash.py via importlib at startup. No registry file. Reads META per specialist to build the format matrix for the GUI.

03 — Specialist

Each *_dash.py defines its own fields, date range, and render config. Data fetched once — rendered to all selected formats. No format logic in the specialist.

04 — Broker Layer

maps/ — routing only. field_map routes Garmin requests to garmin_map. context_map routes to weather_map and pollen_map. No writes, no live API, no domain logic.

05 — Render

Plotters receive complete render config from specialist. dash_layout*.py provides passive resources: colours, tokens, CSS, Plotly CDN cache, template builders.

06 — Output

All output written to BASE_DIR/dashboards/. HTML (interactive Plotly), Excel (openpyxl), JSON + Markdown AI prompt. Format determined by META.output_format.
